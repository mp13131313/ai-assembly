## C26 — ingest/transcription split shipped (2026-05-04 PM, post-C20a)

- New per-session `STATE_NORMALIZED` between `normalizing` and `transcribing` (filesystem source of truth in `01_transcription/<sid>/status.json`).
- `pipeline.run_normalize_and_transcribe` (FastAPI BackgroundTask) renamed to `pipeline.run_normalize` — does ffmpeg only, then sets state=`normalized` and STOPS. Legacy alias preserved so `app.py` calls don't change.
- New `pipeline.fire_transcription(session_dir, session_json_path)` is the orchestrator-facing spawn entrypoint.
- New `overnight_orchestrator.fire_pending_transcriptions(run_dir, session_ids, max_concurrent=4)` scans, counts in-flight, fires up to cap.
- New Stage 0 in `poll_once` runs the dispatcher before the existing transcription_state check. Concurrency cap via `ORCHESTRATOR_TRANSCRIPTION_CONCURRENCY` env var.

**Dashboard:**
- `_transcription_summary` adds `normalized` to counts; new top-level state `awaiting_dispatch` when ALL pending are `normalized` (so operator can see orchestrator-side bottleneck distinctly from ffmpeg or AssemblyAI).
- Per-session label gains "N awaiting dispatch" when normalized > 0.
