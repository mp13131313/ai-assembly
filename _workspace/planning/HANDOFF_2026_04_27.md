## 3. THIS SESSION'S CONTEXT — what I learned

This is the institutional knowledge a fresh session needs to do the document-rewrite work efficiently:

### Session-key insights

1. **Pipeline-fidelity audit method.** When voice_config changes after Phase 0.5 ran, three voice-config fields affect 1a/1b template selection (`type`/`subtype`/`hostile_sources`); the rest only affect Pass 0b tailor. If `hostile_sources` flipped, the dossier was generated WITHOUT the hostile-source warning block and must be re-run from scratch. If only `voice_mode`/`corpus_constraint` changed, only the tailor needs to re-run.

2. **Voice_mode `null` validation.** `node0_validation.py:60` enforces strict 3-enum {philosophical, observational, narratival}. Null only valid for `subtype=system`. Voices with `null` voice_mode were rejected during Phase 0.5 launch this session — Cleopatra and Bob Marley had to be fixed before research could proceed.

3. **Pass 0a precedence.** Hand-authoring voice_configs bypasses the operator-review checkpoint (`03_review_doc.md`). When the operator caught me hand-authoring after voice_config drafts had been written, the right move was to re-run Pass 0a fresh on all 12 voices and accept its verdicts. The hand-authored configs differed from Pass 0a on 3 voices (Cleopatra, Whanganui, Battuta); Pass 0a was sound on each.

4. **FU#49H/I/J/K/D = universal patterns, not Plato-specific edits.** Reviewer pass-3 produced 11 voice-idiom-phrased edits to Plato's persona document. The directive was: capture as universal pipeline edits; the Plato text is the worked example, not the prescription. Voices 3-12 generate their own voice-specific content following the universal instruction.

5. **Position B vs Position C distinction (FU#49D).** Position B = corpus-accurate softening (permit corpus-internal self-criticism). Position C = framework-lifting (permit denying core commitments). The Athens panel format is Position B only — voices speak FROM their frameworks, not against them. FU#49D's universal pattern: hard_limits forbid framework-ABANDONMENT, NOT corpus-internal CROSS-EXAMINATION.

6. **Sentinel_regen smoke-test pattern.** For prompt edits, snapshot pre-edit voice (`<baseline-snapshot-dir>/<voice_slug>/<filename>`), make prompt edit, run `sentinel_regen.py regen --pass <NAME> --voices <slug> --baseline-snapshot <DIR>`, inspect diff. Validate that intended pattern surfaced in regen output. Restore voice baseline post-validation if it was a smoke-test (not a real generation). 5 sentinel runs landed FU#49H/I/J/K/D this week.

7. **FU#50 observations.** (1) Pass 2/4a outputs lack Pydantic enforcement — list-of-string vs list-of-dict shapes both valid per prompts; orchestrator accepts both; this means voices 3-12 will produce inconsistent shapes. **Defer to post-Athens** (regenerating cards mid-prep is risky). (2) sentinel_regen flat-baseline issue — fixed today. (3) Drift on un-edited Pass 2 fields — empirical monitoring directive: watch when voice 3 ships.

8. **The 4 deployment-context JSONs at project root** (per Tier 3): `conference_facts.json` (program), `audience_profile.json` (audience descriptive), `panel_roster.json` (12 voices + casting principle), `council_config.json` (runtime artifact, runtime team owns). Each layer has single ownership. Refreshed for athens-2026 in commit `fc233b8` per `AUDIENCE_BRIEF.md` Part 3 contributors-vs-audience distinction.

### Known traps

- **Don't conflate contributors with audience.** AUDIENCE_BRIEF.md Part 3 line 1: *"The contributors are not the audience. They are what the audience came to encounter."* The 7 intellectual factions describe contributors (~202 speakers); audience (~750 attendees) is more plural than any one faction.
- **Don't over-prime cards via deployment context.** Pass 5 combined deployment priming was 862 words at one point in this session; trimmed to 589. "Brief is key" — the deployment context shapes choices, not output register.
- **Don't put real-person names in the 4 deployment-context JSONs.** Use generic descriptions ("a former PM," "a leading cognitive-warfare theorist," etc.). The voices generate their own engagement_topics; we don't pre-populate with operator-curated targets.
