## What this session landed

### 1. Field-routing refactor (`runtime/flows/voice/card_assembly.py`)

Symmetric extension of what each step's system prompt loads:

- **Voice/expression fields** (rhetorical_mode, characteristic_moves, register_and_tone, metaphorical_repertoire, preferred_vocabulary, banned_language, banned_modes) — now load in Step 1 + 2 + 3 (was 2/3 only). Voice register present from cold-start of detailed reasoning.
- **Reasoning fields** (reasoning_method, finds_compelling, resists, default_questions, disagreement_protocol) — now load in Step 2 (was 1/3 only). Closes the prompt/system mismatch where Step 2's `<decision_1_focus>` cited `finds_compelling`/`resists` as focus discriminators but the fields were not in scope (the unfixed root cause of Test 2 v2's 4/4 woven synthesis bias).

Routing matrix collapsed from 4 distinct sets toward 2: *voice-everywhere* (foundational + reasoning + voice/expression + unique_contribution = 26 fields shared across all steps) + *artifact-only* (8 fields, Step 2/3 only) + narrow step-specific overrides (reference_only_passages Step 1 only; continuity blocks per-step).

Smoke verified across all 4 shipped voices (Plato, Cleopatra, Dostoevsky, Battuta).

### 2. Step 1 + Step 2 prompt rewrites under Haltung lens

Worked through both prompts section-by-section with word-by-word challenge against load-bearing-ness. Each section asks "is this word doing necessary work?" and cuts what isn't.

**Step 1 (`voice_step1_reasoning.md`):** restructured around 9 sections:
```
input → task → your_core → grounding → register → boundaries
      → engaging_the_material → commitment → output
```
- `<your_core>` leads with operational machinery (`reasoning_method`, `default_questions`, `rhetorical_mode`, `characteristic_moves`) on the principle that the closing prompt's job is *activation*, not declaration (declaration happens earlier in the system prompt's IDENTITY/CONSTITUTION/BOUNDARIES sections).
- `default_questions` moved into `<your_core>` (it's a generative field, not a referential one).
- `<grounding>` reduced to `curated_corpus_passages` + `concept_lexicon` (citation/reference tier).
- `<engaging_the_material>` references the STRUCTURED REASONING SURFACE's actual fields (clusters, raw extractions, theme abstract, editorial flags) and makes specific-citation conditional ("where citing specific positions helps your reasoning") rather than mandatory.
- `<commitment>` stripped to climactic vow: *"Reason as only I can. End where my reasoning honestly ends."* (first-person, not orchestrator-frame).

**Step 2 (`voice_step2_artifact.md`):** restructured around 9 sections:
```
input → task → weighing → focus → stance → form → boundaries
      → composition → output
```
- New `<weighing>` block: discrete pre-focus judgment phase with field-grounded first-person introspection (*"Where did `formative_experience` press hardest?"* etc.). Forces the cognitive work of weight-judgment to happen explicitly before focus is committed.
- `<focus>`: conditional binary (one OR synthesis); uncertainty argues for focus, not synthesis. Synthesis must be earned.
- `quality_criteria` invoked as climactic `Pass:` at the end of `<composition>` (was buried as final-check before delivery in original).
- `<composition>` shaped Apply: / Ground: / Pass: triplet — parallel rhythm.
- New `weight_assessment` field added to output schema; parser updated to extract + persist.

Late-session correction: restored the protective instruction "transform the form, carry the substance" on the `relationship_to_detailed_response` bullet in `<composition>` after a reviewer flagged that strong analytical sentences in Step 1 sometimes don't survive into Step 2. Reading the cards' actual `relationship_to_detailed_response` fields revealed the strip is mostly card-mandated (each voice's card explicitly says strip analytical scaffolding, transform analytical → scenic/dramatic/dispositive/fevered). Backstop guards against over-aggressive stripping while honoring the cards' transform-mandate.

### 3. Prefix caching extension (Stage 2 of C19a)

`runtime/flows/voice/card_assembly.py` + `runtime/flows/voice/_anthropic_call.py`:

- `assemble_system_prompt` now returns `(prefix, tail)` tuple. Prefix (~20-25K tokens) is byte-identical across Step 1/2/3 for the same voice/night (name + IDENTITY + CONSTITUTION + BOUNDARIES sections). Tail is step-specific (reference_passages OR artifact + reasoning + voice + continuity + closing prompt).
- `stream_voice_call` accepts `str | tuple[str, str]`; when tuple, places `cache_control` on BOTH blocks (1h TTL).
- New `cache_system: bool = True` kwarg.
- Continuity caller (`continuity.py`) passes `cache_system=False` — single-call flow has no subsequent reads to amortize the 2.0× write penalty.

**Live verification:** Plato Step 1 first call wrote `cache_creation=38,288 tokens`; Step 2 same-voice call read `cache_read=20,086 tokens` from the prefix while writing only `cache_creation=19,665 tokens` for its tail. ✓ Prefix-cache hit confirmed.

## Calibration carried forward

- **Word-by-word challenge protocol works.** Walking each prompt section asking "is this word/clause doing necessary work?" produced both prompts as ~half their previous size while preserving the load-bearing material. Watch for over-aggressive stripping (the reviewer caught one — the `relationship_to_detailed_response` protective backstop). When in doubt, ask "what fails if this clause is removed?"
- **Verify pricing claims before extrapolation.** I cited Opus 4.7 at $15/$75 throughout multiple sessions before fetching the actual docs and finding $5/$25. Always cross-check pricing against platform.claude.com docs for any cost calculation that lands in a planning doc.
- **Field-grounded prompt invocation > generic exhortation.** Step 2's `<weighing>` block uses field-grounded questions ("Where did `formative_experience` press hardest?") rather than generic ones ("Which carried the most pressure?"). Test 3 weight_assessments are substantive specifically because the questions named the discriminating fields.
- **Trust the system prompt's earlier sections.** Closing prompts that repeat what the system prompt already declares add tokens, not signal. Closing prompt's job is to direct application, not declare voice.
- **Live API verification beats theoretical math.** Test 3 cost ~$5-6 vs my theoretical $30-40 estimate. Sanity-check theoretical estimates against actual measured runs whenever feasible.
