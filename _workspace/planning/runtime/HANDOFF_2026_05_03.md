Styled HTML body with "Sign in again" link. Honest footnote about the protocol limitation. Nav adds role-aware logout link, pushed to nav-right via flexbox.

## Pre-Athens calibration carried forward

- **Theme-count envelope revised upward.** v2.4 ratios applied to Athens 13-hour / 15-session / 200-400-extraction night predict **10-18 themes** (possibly 20+ on heterogeneous content), not the 7-10 I'd assumed. Dashboard grids were designed for 60-cell upper bound; Phase C designs should accommodate **up to 100-200 cells** in worst case (Provocateur triage matrix = 10 voices × 18 themes = 180 cells).
- **HTTP Basic realm strings must be ASCII** (RFC 7230). Don't use `·` or `—` or other Unicode punctuation in `WWW-Authenticate` headers.
- **File-open resolver tolerates symlinks**, only rejects `..` traversal. Production VM PROJECT_ROOT is a real directory tree (no symlinks); demo PROJECT_ROOT (`/tmp/c23_demo_*`) symlinks `runs/athens_night_1` → an out-of-tree dryrun. Both work.
- **Producer auth password is in `.env`** (`UPLOAD_APP_PASSWORD=preview` on this machine); admin password is shell-set at server startup. `load_dotenv(override=True)` will overwrite shell-set producer values with `.env` values; this is BY DESIGN (per CLAUDE.md, defends against the Claude Code agent-shell empty-env bug). Demo accordingly used the `.env` value.
- **STATIC_VERSION discipline:** bump on any change to `app.js` or `style.css` (or any template referencing them). Browsers cache `?v=N` aggressively.
