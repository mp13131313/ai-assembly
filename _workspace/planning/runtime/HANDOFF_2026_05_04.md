### Parallelization audit (filed C25, C29, C30)

Audited every flow file for ThreadPoolExecutor usage + sequential for-loops over independent LLM work:

**Already parallel (no gaps):**
- Researcher Node 2 + Node 3: single global LLM calls (atomic by design)
- Provocateur `triage_voice` + `formulate_for_member`: `@task` decorated, `.submit()` queued under Prefect's default ConcurrentTaskRunner
- Voice Step 2 / Step 3 / Continuity: `ThreadPoolExecutor(max_workers=4)`
- Editor: `ThreadPoolExecutor(max_workers=EDITOR_BATCH)` across dossiers
- Publish: file I/O only, no LLM

**Filed gaps (with Athens wall-time savings):**
- **C25** Researcher Node 1 — `for pkg_path in packages: extract_session(...)` sequential. Athens 15 sessions × ~2-3 min = 30-45 min serial → 6-9 min parallel. **~30 min savings/night**.
- **C29** Voice validation — within-voice serial (`for s1 in step1_outputs:`) + within-pair serial (anach then const). Athens bottleneck = max(themes_per_voice) × 2 × ~1 min ≈ 8-10 min → ~3-5 min fully parallel. **~7-10 min savings on Night 1**.
- **C30** Voice Step 1 inter-batch sleep — `time.sleep(VOICE_BATCH_WAIT_S=20)` between batches "to respect rate limits". Athens 30 pairs / 4 = 8 batches → 7 × 20s = ~2.3 min pure wait. Anthropic Tier 4 limits comfortably accommodate; could lower to 5s or 0. **~2 min savings/night**.

**Total addressable Athens wall savings: ~40-45 min** combined across the three.
